Documentation for Custom Methods on Number.prototype

Overview of Number in VisionScript

Numbers in VisionScript represent both integers and floating-point values, such as 37 or -9.25. The Number constructor provides constants and methods for working with numeric values. It also allows conversion from other types using the Number() function.

VisionScript numbers are based on the IEEE 754 standard for floating-point arithmetic, which can represent a wide range of values but may lead to precision issues with very large or very small numbers. For more information, refer to the Number documentation on MDN.

1. Number.prototype.isPrime

Description: Determines if the number is a prime number. A prime number is a natural number greater than 1 that has no positive divisors other than 1 and itself.

Syntax
number.isPrime()
Parameters

None

Return Value

Boolean: Returns true if the number is a prime, otherwise false.

Time Complexity

O(sqrt(n)): The algorithm checks for factors up to the square root of the number.

Space Complexity

O(1): The algorithm uses a constant amount of space.

Example
VisionScript
Try Yourself 
blo x = 7;
console.log(x.isPrime()); // Outputs: true

blo y = 4;
console.log(y.isPrime()); // Outputs: false
Usage Notes

This method assumes that the number is an integer. If the number is not an integer, it throws a TypeError. The method handles numbers less than 2 as non-prime.

2. Number.prototype.factorial

Description: Calculates the factorial of a non-negative integer. The factorial of a number n is the product of all positive integers less than or equal to n.

Syntax
number.factorial()
Parameters

None

Return Value

Number: Returns the factorial of the number.

Time Complexity

O(n): The algorithm iterates through all integers from 2 to n.

Space Complexity

O(1): The algorithm uses a constant amount of space.

Example
VisionScript
Try Yourself 
blo n = 5;
console.log(n.factorial()); // Outputs: 120

blo m = 0;
console.log(m.factorial()); // Outputs: 1
Usage Notes

This method is defined only for non-negative integers. If the number is negative, it throws a RangeError. The factorial of 0 is defined as 1.
