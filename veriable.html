Custom Variable Declarations: blo, glo, and fix


In our new programming language, we introduce custom keywords to declare variables with specific scopes and mutability. These keywords are designed to simplify the declaration process while making the code more readable and intuitive.

VisionScript Variables can be declared in 4 ways:

  • Automatically
  • Using blo
  • Using glo
  • Using fix

1. blo (Block-scoped, Mutable)

Equivalent to: let in JavaScript
Description: Use blo to declare variables that are block-scoped and mutable. These variables are confined to the block in which they are defined and can be reassigned within that block.

VisionScript
Try Yourself 

blo count = 10;
if (count > 5) {
    blo message = "Count is greater than 5";
    console.log(message);  // Output: Count is greater than 5
}
console.log(message);  // Error: message is not defined (because it's block-scoped)

2. glo (Global-scoped, Mutable)

Equivalent to: var in JavaScript
Description: Use glo to declare variables that are globally scoped and mutable. These variables are accessible throughout the entire program and can be reassigned.

VisionScript
Try Yourself 

glo user = "Alice";
function greet() {
    console.log("Hello, " + user);  // Output: Hello, Alice
}
greet();
user = "Bob";
greet();  // Output: Hello, Bob

3. fix (Block-scoped, Immutable)

Equivalent to: const in JavaScript
Description: Use fix to declare variables that are block-scoped and immutable. These variables cannot be reassigned after their initial declaration, ensuring their value remains constant.

VisionScript
Try Yourself 

fix PI = 3.14159;
console.log(PI);  // Output: 3.14159
PI = 3.14;  // Error: Assignment to constant variable
